# 项目架构说明

日期: 2026-04-28

## 项目概述

项目名称: 政策爬虫聚集系统

核心功能:
- **配置化爬虫引擎** - 无需写代码，通过数据库配置即可抓取 HTML 页面
- 多渠道抓取政策文件（RSS、JSON API、Python 脚本、搜索引擎）
- 关键词匹配与政策分发
- 数据管道处理（去重、标准化、匹配、存储）

## 核心模块详解

### 1. HTML 配置化爬虫引擎（新增重点）

**设计理念**: 无需写代码，通过数据库配置即可完成 90% 常见网站的抓取

**核心表结构**:

| 表 | 作用 |
|---|------|
| `policy_data_source` | 数据源配置（类型、Cron 表达式、状态） |
| `policy_crawl_page` | 爬取页配置（列表页/详情页 URL、选择器、超时） |
| `policy_extract_rule` | 字段提取规则（CSS 选择器/JSONPath/正则、属性） |
| `policy_pagination_rule` | 分页规则（URL 模板、页码范围） |

**执行流程（HtmlRunner）**:

```
HtmlCrawlConfigLoader 加载配置
        ↓
buildListUrls() 构建列表页 URL（支持分页）
        ↓
request() 请求页面（Jsoup + 模拟浏览器 UA）
        ↓
select() 提取列表条目元素
        ↓
applyRules() 应用提取规则 → RawDoc
        ↓
[可选] enrichFromDetail() 抓取详情页补充字段
        ↓
shouldSkipByDate() 根据上次抓取时间过滤
        ↓
输出 List<RawDoc>
```

**支持的提取规则**:
- `CSS_SELECTOR` - CSS 选择器（如 `.title a::attr(href)`）
- `JSON_PATH` - JSONPath（如 `$.data.list[*].title`）
- `REGEX` - 正则表达式

---

### 2. 数据源执行器层（sources）

**设计模式**: 策略模式

| 执行器 | 类型 | 说明 |
|-------|------|------|
| `HtmlRunner` | `HTML` | 配置化 HTML 爬虫 |
| `RssRunner` | `RSS` | RSS/Atom 订阅解析 |
| `HttpJsonRunner` | `HTTP_JSON` | JSON API 调用 |
| `PyScriptRunner` | `PYTHON` | 调用 Python 脚本 |
| `SearchRunner` | `SEARCH` | 搜索引擎抓取 |

---

### 3. 数据处理管道（pipeline）

**设计模式**: 管道与过滤器模式

```
RawDoc (原始文档)
    ↓
Deduper (去重，基于 URL 哈希)
    ↓
Normalizer (标准化，清洗日期、机构等)
    ↓
KeywordMatcher (关键词匹配)
    ↓
RawPayloadWriter (存原始数据用于回放)
    ↓
PipelineResult (输出结果)
    ↓
通知观察者（存库、推微信）
```

---

### 4. 数据持久化层（persistence）

**核心实体**:

| 实体 | 作用 |
|-----|------|
| `PolicyDataSource` | 数据源配置（HtmlRunner 的配置入口） |
| `PolicyCrawlPage` | HTML 爬取页配置 |
| `PolicyExtractRule` | HTML 字段提取规则 |
| `PolicyPaginationRule` | HTML 分页规则 |
| `PolicyDocument` | 政策文档（最终结果） |
| `PolicyRawPayload` | 原始载荷（JSON，用于故障排查/数据回放） |
| `PolicyCrawlTaskLog` | 任务执行日志 |

---

### 5. 调度层（scheduler）

**DataSourceScheduler**: 从数据库读取 `PolicyDataSource`，动态注册/删除 Cron 任务

---

## 数据流向

```
定时任务触发 (DataSourceScheduler)
        ↓
RunnerRegistry 获取对应 SourceRunner
        ↓
执行器抓取 (HtmlRunner/RssRunner/...)
        ↓
List<RawDoc>
        ↓
IngestPipeline 管道处理
    ├── Deduper 去重
    ├── Normalizer 标准化
    ├── KeywordMatcher 匹配
    └── RawPayloadWriter 存原始数据
        ↓
PolicySubject 通知观察者
    ├── DatabaseSaveObserver → 存数据库
    └── PushPlusObserver → 推微信
```

## HTML 配置化爬虫使用示例

**场景**: 抓取某省政府官网政策列表

**数据库配置**:

```sql
-- 1. 数据源配置
INSERT INTO policy_data_source (name, type, config, cron_expression, status)
VALUES ('某省政府', 'HTML', '{}', '0 0 * * * ?', 'ACTIVE');

-- 2. 列表页配置
INSERT INTO policy_crawl_page (data_source_id, page_type, url, item_selector, timeout_ms)
VALUES (1, 'LIST', 'http://www.example.gov.cn/policy/list', '.policy-item', 15000);

-- 3. 详情页配置（可选）
INSERT INTO policy_crawl_page (data_source_id, page_type, url, item_selector, timeout_ms)
VALUES (1, 'DETAIL', NULL, '.content', 15000);

-- 4. 提取规则（列表页）
INSERT INTO policy_extract_rule (crawl_page_id, field_name, rule_type, rule_value, priority)
VALUES
(1, 'title', 'CSS_SELECTOR', '.title::text', 1),
(1, 'url', 'CSS_SELECTOR', '.title a::attr(href)', 1),
(1, 'publishDate', 'CSS_SELECTOR', '.date::text', 1),
(1, 'issuingAgency', 'CSS_SELECTOR', '.agency::text', 1);

-- 5. 提取规则（详情页）
INSERT INTO policy_extract_rule (crawl_page_id, field_name, rule_type, rule_value, priority)
VALUES
(2, 'content', 'CSS_SELECTOR', '.content', 1),
(2, 'documentNumber', 'CSS_SELECTOR', '.doc-number::text', 1);
```

## 配置类说明

| 配置类 | 作用 |
|-------|------|
| `JacksonConfig` | 注册 `JavaTimeModule`，支持 `LocalDate`/`LocalDateTime` 序列化 |
| `MybatisPlusConfig` | MyBatis-Plus 分页插件、自动填充、TaskScheduler |

## 设计模式应用

| 模式 | 应用场景 |
|-----|---------|
| **策略模式** | `SourceRunner` 多个实现（HtmlRunner、RssRunner 等） |
| **模板方法** | `HtmlRunner` 中抓取流程骨架 |
| **管道与过滤器** | `IngestPipeline` 数据处理流程 |
| **观察者模式** | `PolicySubject` 通知多个观察者 |
| **门面模式** | `IngestPipeline` 封装复杂处理逻辑 |
| **工厂模式** | `RunnerRegistry` 管理执行器 |
