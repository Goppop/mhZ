下面是一份按当前 `HtmlRunner` 实现整理的 HTML 站点配置文档。现在新增一个 HTML 站点，核心就是配置 4 张表：

`policy_data_source`、`policy_crawl_page`、`policy_extract_rule`、`policy_pagination_rule`。

## 1. 总体关系

一个 HTML 站点配置大致是：

```text
policy_data_source
  -> policy_crawl_page: LIST 列表页
      -> policy_extract_rule: 从每个列表 item 提取 title/url/publishDate 等
      -> policy_pagination_rule: 列表分页规则
  -> policy_crawl_page: DETAIL 详情页，可选
      -> policy_extract_rule: 从详情页提取 content/issuingAgency/documentNumber 等
```

`HtmlRunner` 的执行逻辑是：

1. 根据 `policy_data_source.id` 加载启用的页面配置。
2. 找到第一个 `page_role = LIST` 的页面。
3. 根据分页规则生成列表 URL。
4. 请求列表页，用 `item_selector` 选出每条列表 item。
5. 对每个 item 执行 LIST 页字段提取规则。
6. 如果提取到了 `url`，并且配置了 DETAIL 页，则请求详情页再补充字段。
7. 最终生成 `RawDoc`，后续入库到 `policy_document`。

相关实现主要在 `HtmlRunner.java`、`HtmlCrawlConfigLoader.java`、`FieldExtractor.java`。

## 2. `policy_data_source`：数据源主表

新增站点必须先加一条数据源。

关键字段：

- `name`：数据源名称，例如 `发改委通知公告`。会作为默认 `RawDoc.source`。
- `type`：必须填 `HTML`，否则不会走 `HtmlRunner`。
- `config`：HTML 类型当前不使用，可以填 `NULL`。
- `enabled`：是否启用，`1` 表示启用。
- `cron_expr`：定时抓取表达式，例如 `0 0 8 * * ?`。
- `last_published_at`：增量抓取游标。抓到的 `publishDate` 如果早于它，会被跳过。

示例：

```sql
INSERT INTO policy_data_source (name, type, config, enabled, cron_expr)
VALUES ('某某部门政策公告', 'HTML', NULL, 1, '0 0 8 * * ?');
```

## 3. `policy_crawl_page`：页面配置表

每个 HTML 数据源至少需要一个 `LIST` 页面，`DETAIL` 页面可选。

关键字段：

- `data_source_id`：关联 `policy_data_source.id`。
- `page_role`：页面角色。
    - `LIST`：列表页，必须有。
    - `DETAIL`：详情页，可选。
- `name`：页面配置名称，方便识别。
- `url`：列表页 URL。`DETAIL` 通常为空，因为详情页 URL 从列表 item 里提取。
- `item_selector`：只对 `LIST` 重要，用 CSS 选择器选出每条列表记录。
- `request_method`：当前代码只实现了 `GET`。
- `headers`：当前是预留字段，代码暂未使用。
- `timeout_ms`：请求超时时间，默认建议 `15000`。
- `sort_order`：同类页面排序。加载时取排序靠前的第一个 `LIST` 和第一个 `DETAIL`。
- `enabled`：是否启用。

示例：

```sql
INSERT INTO policy_crawl_page
(data_source_id, page_role, name, url, item_selector, request_method, timeout_ms, sort_order, enabled)
SELECT id, 'LIST', '某某部门列表页',
       'https://example.gov.cn/zcwj/',
       'ul.news-list > li',
       'GET', 15000, 1, 1
FROM policy_data_source
WHERE name = '某某部门政策公告';

INSERT INTO policy_crawl_page
(data_source_id, page_role, name, url, item_selector, request_method, timeout_ms, sort_order, enabled)
SELECT id, 'DETAIL', '某某部门详情页',
       NULL,
       NULL,
       'GET', 15000, 2, 1
FROM policy_data_source
WHERE name = '某某部门政策公告';
```

## 4. `policy_extract_rule`：字段提取规则表

这是最关键的表。每条规则表示“从当前列表 item 或详情页里提取一个字段”。

关键字段：

- `page_id`：关联 `policy_crawl_page.id`。
- `field_name`：要写入 `RawDoc` 的字段名。
- `scope`：当前代码基本不靠它判断，实际是由 `page_id` 属于 LIST 还是 DETAIL 决定提取范围。建议 LIST 写 `ITEM`，DETAIL 写 `PAGE`。
- `selector`：CSS 选择器。为空时表示直接用当前根节点。
- `value_type`：取值方式。
    - `TEXT`：取元素文本。
    - `ATTR`：取元素属性，需要配 `attr_name`。
    - `HTML`：取元素内部 HTML。
    - `CONST`：使用常量，需要配 `const_value`。
    - `REGEX`：从元素文本里正则提取。
- `attr_name`：`value_type = ATTR` 时使用，例如 `href`、`title`。
- `const_value`：`value_type = CONST` 时使用。
- `regex_pattern`：正则提取表达式。若有分组，取第一个分组；否则取完整匹配。
- `date_format`：日期格式，仅 `field_name = publishDate` 时有意义，例如 `yyyy-MM-dd`、`yyyy/MM/dd`、`yyyy年M月d日`。
- `required`：表结构有，但当前代码没有真正强制丢弃失败记录。
- `fallback_group`：表结构有，但当前代码没有按组处理。
- `sort_order`：同一个字段多条规则时的尝试顺序。
- `enabled`：是否启用。

当前支持直接写入 `RawDoc` 的字段名：

- `title`：标题。
- `url`：原文链接。相对路径会基于列表页或详情页 URL 自动补全。
- `source`：来源名称。
- `summary`：摘要。
- `content`：正文。
- `issuingAgency`：发布机构。
- `documentNumber`：文号。
- `publishDate`：发布日期，会解析成日期。
- 其他任意 `field_name`：不会丢，会放到 `metadata` 里。

常见配置：

```sql
-- 列表 item 标题：优先取 a 的 title 属性
INSERT INTO policy_extract_rule
(page_id, field_name, scope, selector, value_type, attr_name, required, sort_order, enabled)
SELECT p.id, 'title', 'ITEM', 'a[href]', 'ATTR', 'title', 1, 1, 1
FROM policy_crawl_page p
JOIN policy_data_source ds ON ds.id = p.data_source_id
WHERE ds.name = '某某部门政策公告' AND p.page_role = 'LIST';

-- 标题兜底：取 a 文本
INSERT INTO policy_extract_rule
(page_id, field_name, scope, selector, value_type, required, sort_order, enabled)
SELECT p.id, 'title', 'ITEM', 'a[href]', 'TEXT', 1, 2, 1
FROM policy_crawl_page p
JOIN policy_data_source ds ON ds.id = p.data_source_id
WHERE ds.name = '某某部门政策公告' AND p.page_role = 'LIST';

-- 原文链接
INSERT INTO policy_extract_rule
(page_id, field_name, scope, selector, value_type, attr_name, required, sort_order, enabled)
SELECT p.id, 'url', 'ITEM', 'a[href]', 'ATTR', 'href', 1, 1, 1
FROM policy_crawl_page p
JOIN policy_data_source ds ON ds.id = p.data_source_id
WHERE ds.name = '某某部门政策公告' AND p.page_role = 'LIST';

-- 列表页发布日期
INSERT INTO policy_extract_rule
(page_id, field_name, scope, selector, value_type, date_format, required, sort_order, enabled)
SELECT p.id, 'publishDate', 'ITEM', '.date', 'TEXT', 'yyyy-MM-dd', 0, 1, 1
FROM policy_crawl_page p
JOIN policy_data_source ds ON ds.id = p.data_source_id
WHERE ds.name = '某某部门政策公告' AND p.page_role = 'LIST';

-- 详情页正文
INSERT INTO policy_extract_rule
(page_id, field_name, scope, selector, value_type, required, sort_order, enabled)
SELECT p.id, 'content', 'PAGE', 'div.article-content, div.TRS_Editor', 'TEXT', 0, 1, 1
FROM policy_crawl_page p
JOIN policy_data_source ds ON ds.id = p.data_source_id
WHERE ds.name = '某某部门政策公告' AND p.page_role = 'DETAIL';
```

## 5. `policy_pagination_rule`：分页规则表

给 LIST 页面配置分页。

关键字段：

- `page_id`：关联 LIST 页的 `policy_crawl_page.id`。
- `mode`：分页模式。
    - `NONE`：不分页，只抓列表页 URL。
    - `URL_TEMPLATE`：按 URL 模板翻页，当前已实现。
    - `NEXT_SELECTOR`：表里有定义，但当前 `HtmlRunner` 尚未实现，会退回只抓第一页。
- `url_template`：`URL_TEMPLATE` 模式使用，例如 `https://example.gov.cn/zcwj/index_{page}.html`。
- `next_selector`：`NEXT_SELECTOR` 预留，当前没用。
- `start_page`：起始页码，默认 `1`。
- `max_pages`：最多抓几页。
- `enabled`：是否启用。

示例：只抓第一页

```sql
INSERT INTO policy_pagination_rule
(page_id, mode, start_page, max_pages, enabled)
SELECT p.id, 'NONE', 1, 1, 1
FROM policy_crawl_page p
JOIN policy_data_source ds ON ds.id = p.data_source_id
WHERE ds.name = '某某部门政策公告' AND p.page_role = 'LIST';
```

示例：URL 模板分页

```sql
INSERT INTO policy_pagination_rule
(page_id, mode, url_template, start_page, max_pages, enabled)
SELECT p.id, 'URL_TEMPLATE',
       'https://example.gov.cn/zcwj/index_{page}.html',
       1, 5, 1
FROM policy_crawl_page p
JOIN policy_data_source ds ON ds.id = p.data_source_id
WHERE ds.name = '某某部门政策公告' AND p.page_role = 'LIST';
```

## 6. 配置新站点时的最小必填项

最小能跑起来，需要这些：

- `policy_data_source`：一条 `type = HTML`、`enabled = 1` 的数据源。
- `policy_crawl_page`：一条 `page_role = LIST` 的列表页，必须有 `url` 和 `item_selector`。
- `policy_extract_rule`：LIST 页至少配置：
    - `title`
    - `url`
    - 最好再配 `publishDate`
- `policy_pagination_rule`：可以配 `NONE`。
- 如果想抓正文，再加：
    - 一条 `page_role = DETAIL` 页面。
    - DETAIL 页的 `content` 提取规则。

## 7. 实操建议

新增站点时，先在浏览器 DevTools 里确认 3 个选择器：

- 列表 item：例如 `ul.news-list > li`
- item 内标题/链接：例如 `a[href]`
- 详情页正文：例如 `div.article-content`

然后先只配 LIST，确认能抓到 `title` 和 `url`；再加 DETAIL 补 `content`、`issuingAgency`、`documentNumber`。当前这版最容易出问题的地方通常是 `item_selector` 选太大或太小，以及 `publishDate` 的日期格式不匹配。
